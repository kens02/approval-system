## 3. 業務フロー

### 3.1 休暇申請の典型フロー

```
[申請者]                [ステップ1]      [ステップ2]      [決裁者]        [時間管理者]
   │                       │                │                │                │
   ├─ 申請作成 (draft)     │                │                │                │
   ├─ 提出 ───────────────►│ メール通知     │                │                │
   │                       ├─ 承認 ────────►│ メール通知     │                │
   │                       │                ├─ 承認 ────────►│ メール通知     │
   │                       │                │                ├─ 決裁 ─────────►│ 通知
   │                       │                │                │                │
   │                       │                │                │  ┌─────────────┤ クローズ
   │ ◄──────── 完了通知 ──────────────────────────────────────┘  ▼
   │                                                            closed
```

### 3.2 差し戻しフロー

```
[ステップN]               [元申請A]                      [新申請B]              [申請者]
   │                         │                              │                      │
   ├─ 差し戻し（コメント必須）│                              │                      │
   │                         ├─ status=rejected             │                      │
   │                         ├─ is_deleted=true             │                      │
   │                         │                              │                      │
   │                         │ 内容コピー                   │                      │
   │                         ├─────────────────────────────►│ status=draft         │
   │                         │ parent_application_id ───────┤                      │
   │                         │                              ├─ メール通知 ────────►│ 修正・再提出
   │                         │                              │                      │
```

### 3.3 取り消しフロー

申請者（in_progress 中）/ 時間管理者 / システム管理者が実行可能。`is_deleted=true, status=cancelled` とし、関与済みステップ担当者と時間管理者へ通知する。

## 5. 機能設計

### 5.1 申請ステータス遷移ルール

| 現ステータス | アクション | 実行ロール | 次ステータス | 備考 |
|---|---|---|---|---|
| draft | submit | 申請者 / 時間管理者 | in_progress | step=1 へ通知 |
| in_progress | approve | 当該ステップ担当 / 時間管理者 | in_progress (step+1) | 最終ステップなら decide へ |
| in_progress | decide | 最終ステップ担当 / 時間管理者 | decided | 申請者＋時間管理者通知 |
| in_progress | reject | 当該ステップ担当 / 時間管理者 | rejected (soft-delete) | 新申請を draft で複製 |
| in_progress | cancel | 申請者 / 時間管理者 | cancelled (soft-delete) | 関与者へ通知 |
| decided | close | 時間管理者 / システム管理者 | closed | 申請者へ完了通知 |

### 5.2 残日数計算（疑似コード）

```
function calcRemaining(userId, asOfDate):
    s = SystemSettings(tenant)
    fyStart = fiscalYearStart(asOfDate, s.fiscal_year_start_month)
    
    carried = LeaveBalances[year=fyYear, month=s.fiscal_year_start_month].carry_over_days
    granted = sum(LeaveBalances WHERE year/month BETWEEN fyStart AND asOfDate).granted_days
    used    = sum(approved LeaveApplications IN current FY).total_minutes / s.work_minutes_per_day
    
    return (carried + granted) - used  // 単位: 日（小数 = 分換算）
```

繰越:
```
on fiscal year close:
    carryOver = min(remainingDays, max_carryover_days)
    new LeaveBalances(year=newFY, month=startMonth, carry_over_days=carryOver)
```

### 5.3 申請番号採番

- フォーマット: `yyyy-mm-NNNN`
- スコープ: テナント × 年月
- 実装: `ApplicationNumberGenerator` サービスがトランザクション内で `MAX(NNNN)+1` を取得（行ロック必須）

### 5.4 通知ルール

| イベント | 受信者 | 件名例 |
|---|---|---|
| submitted | step1 担当者 | 【承認依頼】休暇申請（{申請者氏名}） |
| step_approved | step N+1 担当者 | 【承認依頼】（次ステップ） |
| rejected | 申請者 | 【差し戻し】（コメント） |
| decided | 申請者 + テナント時間管理者全員 | 【決裁完了】クローズ処理をお願いします |
| closed | 申請者 | 【完了通知】 |
| cancelled | 関与済みステップ担当者全員 + 時間管理者 | 【取り消し】 |

## 8. 印刷設計

### 8.1 一覧表（A4横）
- 1ページ 25 件目安
- ヘッダ: 組織名・出力日時・期間
- フッタ: ページ番号

### 8.2 定型用紙（A4縦）
- 申請者欄 / 承認フロー欄（印鑑欄付き）/ 申請内容欄
- 決裁日・決裁者氏名は実データから差し込み

### 8.3 実装方式
- 第一候補: HTML + `@media print` CSS + `window.print()`
- PDF が要件化された場合: QuestPDF（C# サーバ生成）
