## 3. 業務フロー

### 3.1 標準フロー（申請→決裁→クローズ）

```
[申請者]                  [ステップ1]     [ステップ2]     [決裁者]      [環境設定者]
   │                          │                │              │              │
   ├─ 定型用紙シートに記入    │                │              │              │
   ├─ 「提出」ボタン押下       │                │              │              │
   │   ↓ 決裁パスワード入力    │                │              │              │
   ├─ 申請リストへ追記         │                │              │              │
   ├─ メール送信 ──────────► │                │              │              │
   │                          ├─ ファイルを開く │              │              │
   │                          ├─ 「承認」ボタン押下           │              │
   │                          │   ↓ 決裁パスワード             │              │
   │                          ├─ メール送信 ──► │              │              │
   │                          │                ├─ 承認 ──► │              │
   │                          │                │             ├─ 決裁         │
   │                          │                │             │  ↓決裁パスワード│
   │                          │                │             ├─メール─────► │
   │                          │                │             │             ├─ クローズ操作
   │                          │                │             │             │  ↓ 環境設定パスワード
   │ ◄──────────── 完了通知 ──────────────────────────────────────┘
```

### 3.2 差し戻しフロー

```
[ステップN]                [申請リスト]                       [申請者]
   │                            │                                │
   ├─「差し戻し」ボタン押下      │                                │
   ├─ コメント入力（必須）       │                                │
   ├─ 決裁パスワード             │                                │
   │                            │                                │
   ├─ 元行 status=rejected,     │                                │
   │   is_deleted=true へ更新   │                                │
   ├─ 内容コピー                 │                                │
   │   ↓                         │                                │
   │   新行 status=draft         │                                │
   │   parent_application_id=元 │                                │
   ├─ メール送信 ──────────────────────────────────────────────►│
```

### 3.3 取り消しフロー

申請者本人（in_progress 中）または環境設定者が実行可能。  
対象行を `is_deleted=true, status=cancelled` に更新し、関与済み担当者と環境設定者に通知。

## 6. 申請ステータス遷移

| 現在 | アクション | 実行ロール | 次状態 | 備考 |
|---|---|---|---|---|
| draft | submit | 申請者 | in_progress | step=1 へ通知 |
| in_progress | approve | 当該ステップ担当 / 環境設定者 | in_progress (step+1) | 最終ステップなら decide |
| in_progress | decide | 最終担当 / 環境設定者 | decided | 申請者+環境設定者通知 |
| in_progress | reject | 当該担当 / 環境設定者 | rejected (soft-delete) | 新行 draft で複製 |
| in_progress | cancel | 申請者 / 環境設定者 | cancelled (soft-delete) | 関与者通知 |
| decided | close | 環境設定者 | closed | 申請者通知 |

## 7. メール通知設計

### 7.1 送信タイミング

| イベント | 受信者 |
|---|---|
| 提出（submitted） | ステップ1担当者 |
| 承認（step_approved） | 次ステップ担当者 |
| 差し戻し（rejected） | 申請者 |
| 決裁完了（decided） | 申請者 + 環境設定者全員 |
| クローズ（closed） | 申請者 |
| 取り消し（cancelled） | 関与済み担当者全員 + 環境設定者 |

### 7.2 メール本文への必須要素

- 申請番号 / 申請者氏名 / 取得日時 / フェーズ名
- **共有フォルダの Excel ファイル UNC パス**（クリックで Excel が起動）
- 該当申請の `application_id`（行特定用）
- 差し戻しの場合: コメント

### 7.3 送信方式

- **第一候補**: Outlook COM 自動化（`Outlook.Application` 経由で送信トレイに投入）
- **代替**: CDO（`CDO.Message`）+ SMTP（`環境設定` シートに SMTP 情報保存）
- 失敗時: `メール送信ログ` に `failed` を記録、後で再送可能

## 8. アクセス制御方針（基本）

詳細は `access-control.md`。基本方針のみ記載。

| 制限 | 認証方式 | 対象操作 |
|---|---|---|
| ファイル閲覧 | Excel 標準のブック開封パスワード | Book を開く |
| 決裁 | `環境設定` シートに格納の決裁パスワード | 承認・決裁・差し戻し・取り消し |
| 環境設定 | `環境設定` シートのシート保護パスワード | マスタ・設定の編集、クローズ処理、代行決裁 |

すべてのシート編集は VBA を経由する設計とし、シート保護を解除→処理→再保護の流れで一貫させる。

## 9. ロックファイル方式（同時編集抑止）

```
Book を開く
   │
   ├ \\share\承認決裁\.lock 存在チェック
   │    │
   │    ├ 存在する場合 → 「他のユーザーが編集中です」警告 → 読み取り専用で開く
   │    └ 存在しない場合 → .lock 作成（自分のユーザー名・タイムスタンプ）
   │
   └ Book を閉じる時 → .lock 削除
```

- ロックファイルは UTF-8 テキスト（`username:timestamp`）
- 異常終了時に `.lock` が残存することがあるため、環境設定者は強制解除メニューを持つ

## 12. 起動時／終了時処理

### 12.1 Workbook_Open（起動時）

```
1. ロックファイルチェック
2. マクロが有効でない場合は警告（このコードは実行されないため、シート上にメッセージ表示）
3. 環境設定の読み込み（一部キャッシュ）
4. Excel バージョンチェック
5. ロックファイル生成
6. ダッシュボードシートをアクティブ化
```

### 12.2 Workbook_BeforeClose（終了時）

```
1. 未送信メールキューの確認（任意）
2. ロックファイル削除
3. 自動バックアップ（任意・オプション）
```
