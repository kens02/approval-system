# Excel VBA 暫定ツール 要件定義書

**バージョン**: 1.0  
**作成日**: 2026-05-01  
**親要件**: `要件定義書.md`（Web 版 v1.2）

---

## 1. 目的・位置づけ

Web 版承認決裁システム稼働までの **つなぎ運用** として、Excel VBA で休暇申請ワークフローを電子化する。Web 版の要件定義書（v1.2）を踏襲しつつ、以下の方針で **縮約** する。

| 項目 | Web 版 | 暫定ツール |
|---|---|---|
| マルチテナント | あり | **なし**（1ファイル=1組織/個人運用） |
| ロール | 6種 | **3層**（閲覧者 / 申請者・決裁者 / 環境設定者） |
| 認証 | JWT | **Book／シート保護パスワード** |
| 申請種別 | 休暇申請（年次有給） | 同左 |
| ワークフロー | 同左 | 同左（リレー方式・差し戻し・取消・クローズ） |
| メール通知 | MailKit | **Outlook COM 自動化**（CDO 代替） |
| 印刷 | HTML/CSS or PDF | Excel 印刷（A4縦・定型用紙） |

---

## 2. 機能要件（Web 版からの差分）

### 2.1 採用する機能

- 休暇申請（年次有給休暇のみ）
- リレー方式の承認フロー（合議 → 承認 → 決裁）
- 差し戻し（行ソフトデリート＋複製＋ステップ1から再開）
- 取り消し（ソフトデリート）
- 時間管理者（=環境設定者）によるクローズ処理
- メール自動通知（提出/承認/差し戻し/決裁/クローズ/取消）
- 申請番号 `yyyy-mm-NNNN`
- 休暇取得単位: 時間（15分単位）/ 日（全休・午前・午後）
- 月次付与・年度繰越・上限切捨て
- 印刷（定型用紙・一覧表）
- 操作ログ（監査）

### 2.1.1 デバッグモード（開発・動作確認用）

開発時の検証効率を確保するため、**デバッグモード** を提供する。

| 項目 | デバッグモード時の挙動 |
|---|---|
| 決裁/環境設定パスワード | 要求しない（自動 OK） |
| シート保護 | 解除したまま（再保護しない） |
| 隠しシート | 全シートを表示（手動で再非表示にも対応） |
| メール送信 | 実送信せず、ダイアログで内容を表示 |
| 操作ログ | 記録するが `is_debug=TRUE` フラグを立てる（Web 版インポート時は既定で除外） |
| 切り替え方法 | 環境設定パスワードで認証してトグル |
| ファイル閲覧パスワード | デバッグモードでも変更しない（Excel 標準のため VBA で制御不可） |

> 詳細仕様は `debug-mode.md` 参照。配布前に必ず OFF にすること。

### 2.2 簡略化・除外する機能

| 機能 | 扱い | 理由 |
|---|---|---|
| マルチテナント | 除外 | 1ファイル=1組織で運用 |
| システム管理者と時間管理者の分離 | **統合**（=環境設定者） | 個人ツールのため運用簡素化 |
| 二要素認証 | 除外 | パスワード保護で代替 |
| AD/LDAP 連携 | 除外 | 本ツール不要 |
| API・外部統合 | 除外 | スタンドアロン |
| 統計ダッシュボード | **簡易版**（任意シート） | 後回し可 |
| 添付ファイル | 除外 | 後続検討 |
| GUI（カード入力フォーム） | **設計のみ**・実装は後続 | 初期は **シート直入力** |

### 2.3 ロール定義（3層）

| ロール | 該当者 | 権限 |
|---|---|---|
| **閲覧者** | 申請対象外の社員、関係者 | Book を開いて閲覧（Bookパスワード必要）。決裁・編集は不可 |
| **申請者・決裁者** | 申請する本人、承認者、決裁者 | 申請作成、自担当ステップでの承認/差し戻し（**決裁パスワード**で認証） |
| **環境設定者**（=Web版「システム管理者+時間管理者」） | 管理者本人 | 環境設定・マスタ編集・代行決裁・クローズ処理・残日数修正（**環境設定パスワード**で認証） |

## 3. 非機能要件

### 3.1 セキュリティ

- **3層パスワード方式**（詳細: `access-control.md`）
- パスワードは **`環境設定` シートに集約**（VBAから参照）
- すべてのシートは VBA 以外で編集不可とし、特権操作は VBA でシート保護を一時解除する
- 操作ログは VBA 以外で編集不可とし、改ざん検知のためハッシュ列を付与（任意）

### 3.2 性能

- 想定申請件数: 5 年で **10,000 件以下**（個人/部門単位）
- 起動時間: 10 秒以内（Workbook_Open のチェック含む）
- 一覧表示: 5,000 件で 3 秒以内

### 3.3 可用性

- 共有フォルダ上で運用、日次バックアップ（運用設計）
- 同時編集は想定しない（**ロックファイル**方式で抑止）
- マクロ無効環境では **読み取り専用** で動作（決裁機能は停止）

### 3.4 互換性・前提

- **Microsoft Excel 2019 以上 / Microsoft 365**
- ファイル形式: `.xlsm`（マクロ有効）
- 文字コード: UTF-8（CSV エクスポート時）
- メール: Outlook 2019 以上（COM 自動化）。Outlook がない環境では CDO で代替

## 5. 通知要件

Web 版と同等のタイミングで送信する。

| イベント | 受信者 | 件名（テンプレ既定） |
|---|---|---|
| submitted | ステップ1担当者 | 【承認依頼】休暇申請 - {申請者氏名} |
| step_approved | ステップN+1担当者 | 【承認依頼】休暇申請 - {申請者氏名}（{フェーズ名}） |
| rejected | 申請者 | 【差し戻し】休暇申請 - {コメント} |
| decided | 申請者 + 環境設定者 | 【決裁完了】休暇申請 - クローズ処理願います |
| closed | 申請者 | 【完了通知】休暇申請 |
| cancelled | 関与済みステップ担当者全員 + 環境設定者 | 【取り消し】休暇申請 |

通知本文には **共有フォルダ上の Excel ファイルへの UNC パス**（例: `\\fileserver\share\承認決裁\hr-2026.xlsm`）を必ず付記する。

## 7. 制約

| 項目 | 制約 |
|---|---|
| 同時編集 | 不可（ロックファイル方式で排他） |
| ファイルサイズ | 50MB を超えない範囲で運用（超過時は年度別ファイル分割） |
| 添付ファイル | 非対応 |
| 外部API | 利用しない |
| マクロ無効時 | 読み取りのみ（編集・決裁不可） |

## 8. 受入基準（抜粋）

- [ ] Book を開くとパスワードが要求される
- [ ] 環境設定シート編集には環境設定パスワードが必要
- [ ] 決裁操作には決裁パスワードが必要
- [ ] 申請提出後、ステップ1担当者にメールが届き、共有フォルダのファイルパスが記載されている
- [ ] リレー方式で順次次の担当者に通知される
- [ ] 差し戻し時に元行はソフトデリートされ、複製行が新規申請として作成される
- [ ] 決裁完了後、環境設定者がクローズ操作で `closed` に遷移できる
- [ ] 操作ログに全状態遷移が記録される
- [ ] CSV エクスポートで Web 版に移行可能な形式で出力される
- [ ] デバッグモードへ切り替えると、パスワード認証・メール送信・シート保護が緩和される
- [ ] デバッグモード解除で運用既定の保護状態に戻る

## 9. 用語

Web 版要件定義書の用語に準拠。本ツール固有の用語は以下。

| 用語 | 定義 |
|---|---|
| **環境設定者** | Web 版の「システム管理者+時間管理者」を統合した本ツール固有のロール |
| **決裁パスワード** | 承認/決裁/差し戻し操作の認証用パスワード |
| **環境設定パスワード** | 環境設定シート・マスタ編集時の認証用パスワード（最高権限） |
| **共有フォルダ** | Book の配置場所。UNC パス（例: `\\fileserver\share\...`）を指す |
