# 承認決裁システム ─ Excel VBA 暫定ツール 設計書一式

**バージョン**: 1.0  
**作成日**: 2026-05-01  
**位置づけ**: 本プロジェクト（Web 版）の運用開始までの **暫定ツール（個人単位運用）**

---

## 1. 本ツールの目的・スコープ

| 項目 | 内容 |
|---|---|
| 目的 | Web 版承認決裁システムが本稼働するまでの **つなぎ運用** として、休暇申請ワークフローを Excel + VBA で実現する |
| 利用形態 | **個人単位で 1 ファイルを管理**（部門ごと・申請者ごとに Book を分ける運用も可） |
| 配置 | 共有フォルダ（社内ファイルサーバ）に配置し、Book パスワードでアクセス制限 |
| ロール簡略化 | マルチテナントは持たない。ロールは「**閲覧者 / 申請者・決裁者 / 環境設定者（=システム管理者+時間管理者）**」の3階層に簡略化 |
| 開発分担 | VBA ソースコードは Claude Code で整備、Excel ファイルへの取り込み・テストは **手作業** |
| 将来 | Web 版稼働時に Excel 内データを **インポート** できるよう、シート構造・キー設計を Web 版データモデルに揃える |

## 3. シート構成（全体像）

ユーザー指示の基準4シート（定型用紙・決裁ルート・申請リスト・環境設定）に加え、以下を提案する。

| # | シート名 | 役割 | 保護種別 | 備考 |
|---|---|---|---|---|
| 1 | `定型用紙` | 申請書フォーム（印刷出力・新規申請の起点） | 入力欄以外をシート保護 | A4縦・印鑑欄 |
| 2 | `申請リスト` | 全申請の台帳（メインデータ）  | 行追加のみ可・編集はVBA経由 | Web版 `Applications` 相当 |
| 3 | `決裁ルート` | 申請者ごとのルート（合議/承認/決裁・担当者） | 決裁パスワード保護 | Web版 `ApprovalRoutes` 相当 |
| 4 | `環境設定` | パスワード3種・SMTP・共有フォルダパス等 | **環境設定パスワード**で保護 | 最重要シート |
| 5 | `ユーザーマスタ` | 氏名・メール・ロール | 環境設定パスワード保護 | メール宛先・ルート設定参照元 |
| 6 | `メールテンプレート` | 通知文面（提出/承認/差し戻し/決裁/クローズ/取消） | 環境設定パスワード保護 | プレースホルダ対応 |
| 7 | `休暇残管理` | 月次付与・繰越・取得分（個人単位） | 環境設定パスワード保護 | Web版 `LeaveBalances` 相当 |
| 8 | `操作ログ` | 全操作の監査ログ（誰がいつ何をしたか） | VBA 以外編集不可 | 改ざん防止 |
| 9 | `メール送信ログ` | 送信履歴 | VBA 以外編集不可 | 失敗時の再送元 |
| 10 | `ダッシュボード` | 残日数・申請中件数・統計（簡易） | 計算式のみ・全保護 | 任意（後続追加可） |

> シート1〜4が **必須**、5〜9が **推奨**、10が **任意**。

## 4. アクセス制御方針（要約）

3 層のパスワードを **`環境設定` シート内に集約保管** し、すべて VBA 経由でしか参照できない構成にする。

| 制限種別 | 対象 | パスワード保管場所 | 認証タイミング |
|---|---|---|---|
| **ファイル閲覧制限** | Excel Book を開く | （個別管理。Excel 標準のブック開封パスワード） | Book を開いた瞬間 |
| **決裁パスワード制限** | 承認・決裁・差し戻し操作 | `環境設定` シート（環境設定パスワード保護下） | 決裁ボタン押下時 |
| **環境設定パスワード制限** | 環境設定・マスタ編集 | （個別管理。VBA で `Sheets.Unprotect` 時に使用） | 設定シート編集／VBA特権処理時 |

> 詳細: [access-control.md](access-control.md)

## 4.1 デバッグモード（開発・動作確認用）

開発時に保護機構（パスワード認証・シート非表示・メール送信）が検証を妨げないよう、**デバッグモード** を提供する。

| 切替時 | 動作 |
|---|---|
| 切替方法 | 環境設定パスワードで認証 → トグル |
| 認証 | 決裁/環境設定パスワードを **要求しない** |
| シート保護 | **解除されたまま**（再保護しない） |
| 隠しシート | 全シートを表示（手動で再非表示も可） |
| メール送信 | 実送信せず **ダイアログ表示のみ** |
| 操作ログ | `is_debug=TRUE` で記録 |

> 詳細: [debug-mode.md](debug-mode.md)

## 5. 開発・展開フロー

```
┌────────────────────┐    ┌────────────────────┐    ┌────────────────────┐
│ Claude Code で VBA │ →  │ 手作業で Excel に  │ →  │ 手作業でテスト・     │
│ コード・設計を整備  │    │ モジュール取り込み │    │ シート保護・配布     │
└────────────────────┘    └────────────────────┘    └────────────────────┘
         ↑                                                     │
         └─────────────── 修正フィードバック ──────────────────┘
```

- VBA ソースは `src/excel_vba/` 配下にエクスポート形式（`.bas` / `.cls` / `.frm`）で保管予定
- 取り込みは VBE で「ファイル → ファイルのインポート」、または `VBComponents.Import` を使ったマクロで一括投入

## 6. Web 版との関係

| 項目 | 暫定ツール（Excel） | Web 版 |
|---|---|---|
| マルチテナント | なし（1ファイル=1テナント相当） | あり |
| ロール数 | 3層（閲覧 / 決裁 / 環境設定） | 6種類 |
| 認証 | Book／シート保護パスワード | JWT |
| ステータス | 同等（draft/in_progress/decided/closed/rejected/cancelled） | 同等 |
| 申請番号 | `yyyy-mm-NNNN` ※同フォーマット | `yyyy-mm-NNNN` |
| メール通知 | VBA + CDO/Outlook 自動化 | MailKit |
| 監査ログ | 操作ログシート | `ApprovalStepLogs` |
| データ移行 | CSV/Excel エクスポート → Web版インポートAPI | - |

> 申請番号フォーマット・ステータス値・主要カラム名は Web 版に揃え、移行時の変換ロスを最小化する。

## 7. 制約・前提条件

- 対象 Excel: **Microsoft Excel 2019 以上 / Microsoft 365**
- マクロ有効（`.xlsm` 形式）
- 共有フォルダへのアクセス権は社内ポリシーに従う
- 同時編集は想定しない（**1人ずつ開いて編集** ─ 共有Bookモードは使わない）
- 添付ファイルは初期リリースでは非対応
- メール送信: Outlook 自動化（COM）を第一候補、CDO を代替候補

---

## 8. 既知のリスクと緩和策

| リスク | 緩和策 |
|---|---|
| 同時編集による衝突 | 編集前にロックファイル `.lock` を作成し、開けない場合は読み取り専用で開く |
| パスワード漏洩 | 環境設定者のみが知るパスワードを定期的に更新する運用ルールを定める |
| マクロ無効環境 | 起動時に `Workbook_Open` でマクロ有効化を案内、無効時は決裁機能を停止 |
| ファイル破損 | 共有フォルダで日次バックアップ（運用） |
| 監査ログ改ざん | `操作ログ` シートを VBA 以外編集禁止＋ハッシュ列を付与（任意） |
